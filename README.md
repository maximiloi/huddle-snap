# 🏈 HuddleSnap v3.6

**Таймер хаддла и снэпа для флаг-футбола**

PWA-приложение для управления временем хаддла, отсчёта снэпа и подсчёта даунов во время игры в флаг-футбол.

---

## ✨ Возможности

### 🎯 Основные функции

- **Таймер хаддла** — настраиваемое время от 25 до 45 секунд (шаг 5 сек)
- **Отсчёт снэпа** — 7-секундный обратный отсчёт перед снэпом
- **Счётчик даунов** — автоматический подсчёт попыток (1st, 2nd, 3rd, 4th)
- **Кнопка нарушения** — управление повтором или потерей дауна
- **Звуковые сигналы** — свисток, сирена, голосовое объявление "10 секунд"
- **Вибрация** — тактильная обратная связь (на поддерживаемых устройствах)

### 📱 PWA-функции

- **Офлайн-работа** — приложение работает без интернета после первой загрузки
- **Установка на главный экран** — работает как нативное приложение
- **Адаптивный дизайн** — оптимизировано для мобильных устройств
- **Крупные кнопки** — удобно использовать в перчатках на поле

---

## 🎬 Сценарий использования

1. **Выбор времени хаддла** → 25/30/35/40/45 секунд
2. **"Время запущено"** → звучит свисток → начинается отсчёт
3. **На 10 секунде** → голосовое объявление "10 секунд" + вибрация
4. **Нажатие "Снэп"** → таймер останавливается → 7-секундный отсчёт
5. **После снэпа** → сирена → даун +1 → возврат к готовности
6. **Нарушение** → модальное окно с выбором: повтор или потеря дауна

## 🚀 Установка и запуск

### Локальная разработка

1. Клонируйте репозиторий:

```bash
git clone https://github.com/your-username/huddle-snap.git
cd huddle-snap
```

2. Откройте `index.html` в браузере или запустите локальный сервер:

```bash
# Python 3
python -m http.server 8000

# Node.js (npx)
npx serve
```

3. Откройте в браузере: `http://localhost:8000`

### Развёртывание на GitHub Pages

1. Создайте репозиторий на GitHub
2. Загрузите все файлы
3. Перейдите в **Settings → Pages**
4. Выберите **Source: Deploy from a branch**
5. Выберите ветку `main` и папку `/ (root)`
6. Через минуту приложение будет доступно по адресу:
   `https://your-username.github.io/huddle-snap/`

### Установка как PWA

**Android (Chrome):**

- Откройте приложение в браузере
- Нажмите на меню (⋮) → "Добавить на главный экран"

**iOS (Safari):**

- Откройте приложение в Safari
- Нажмите "Поделиться" (⬆️) → "На экран «Домой»"

## 🎵 Звуковые файлы

Приложение использует три MP3-файла:

### whistle.mp3

- **Длительность:** ~1.0–1.5 сек
- **Описание:** Одиночный свисток судьи
- **Где взять:** [Freesound.org](https://freesound.org/search/?q=referee+whistle)

### siren.mp3

- **Длительность:** ~3.0–4.0 сек
- **Описание:** Сирена окончания времени
- **Где взять:** [Freesound.org](https://freesound.org/search/?q=game+siren)

### ten_seconds.mp3

- **Длительность:** ~1.0–1.5 сек
- **Описание:** Голосовое объявление "10 секунд"
- **Как создать:** Запишите голосом или используйте [TTS-сервис](https://ttsmaker.com/)

**Важно:** Если MP3-файлы не загружены, приложение автоматически использует синтезированные звуки (fallback).

## 🎨 Интерфейс

### Экран настройки

- Счётчик даунов сверху (1st, 2nd, 3rd, 4th)
- Выбор времени хаддла
- Кнопка "Продолжить"

### Экран готовности

- Счётчик даунов сверху
- Большая зелёная кнопка "Время запущено"
- Кнопка "Нарушение" (появляется после первого драйва)

### Экран работы таймера

- Счётчик даунов сверху
- Большой таймер (меняет цвет: белый → жёлтый → красный)
- Статус ("Хаддл" / "Снэп через N")
- Три кнопки: "Снэп", "2 КБ", "Нарушение"

## 🔧 Технические детали

### Технологии

- **HTML5** — структура
- **CSS3** — стили (flexbox, градиенты, анимации)
- **JavaScript (ES6+)** — логика приложения
- **Web Audio API** — воспроизведение звуков

- **Service Worker** — офлайн-кэширование

- ✅ Firefox
